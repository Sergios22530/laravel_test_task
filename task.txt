Технічне завдання
Мета:
Розробити міні-систему управління завданнями (To-Do List) на Laravel. Основний функціонал: авторизація користувачів, створення, редагування, видалення і перегляд завдань..;
Вимоги до функціоналу:
Авторизація користувачів
Використати стандартну аутентифікацію Laravel.
Додати можливість скидання пароля через email.
Робота із завданнями (CRUD-операції)
Створення завдання: поля — заголовок (обов’язкове), опис (опціонально), статус (за замовчуванням "в очікуванні"), дата завершення (опціонально), пріоритет (низький, середній, високий).
Перегляд завдань: вивести список усіх завдань користувача з пагінацією (по 10 завдань на сторінці).
Редагування завдання.
Видалення завдання.
Деталі завдання: окрема сторінка з інформацією про завдання.
Фільтрація та сортування завдань
Фільтрувати за статусом ("в очікуванні", "в процесі", "завершена") і пріоритетом.
Сортувати за датою створення та завершення.
API для роботи із завданнями
Реалізувати CRUD-операції через REST API з захистом через API Token Authentication.
Основні endpoints:
GET /api/tasks – отримати список завдань з фільтрацією та сортуванням.
POST /api/tasks – створити нове завдання.
PUT /api/tasks/{id} – оновити завдання.
DELETE /api/tasks/{id} – видалити завдання.
GET /api/tasks/{id} – отримати деталі завдання.
Тестування
Написати тести для основних функцій: створення, редагування, видалення завдань та аутентифікації.
Вимоги до реалізації:
Laravel 10-1.
Використання MVC.
Міграції для роботи з базою даних.
Валідація даних при створенні та редагуванні завдань.
Eloquent ORM.
Laravel Resource для форматування даних в API.
Рекомендації:
Придержуватись PSR-12 стандартів.
Використовувати Git для контролю версій (приватний репозиторій на GitHub або GitLab).

Можливість завантаження та відображення файлів (вкладень до завдань).
README.md з інструкціями для запуску проєкту.
Docker для швидкого розгортання.
